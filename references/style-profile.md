# Style Profile

Use this as the generic fallback writing reference for strategy reports. Treat it as a flexible profile, not a rigid template. The skill's default house voice lives in [style-presets/analytical-operator.md](style-presets/analytical-operator.md). Use this file only when the user wants a more neutral strategy-report voice or when no preset fits.

## Override rules

- Let user-supplied style rules, sample text, or explicit writing constraints override this default profile.
- Keep this priority order:
  1. Accuracy and explicit boundaries
  2. User-supplied style constraints
  3. Selected preset
  4. Default style profile
- If the user gives only partial style input, merge it into this default profile instead of discarding the rest.

## Language style

- Write in a calm, professional, result-oriented tone.
- Use business and strategy terms, but control jargon density.
- Prefer short definition-style sentences such as “核心是”, “本质是”, “关键前提是”, “这意味着”.
- Use numbers, ranges, and explicit tradeoffs when the evidence supports them.
- Explain terminology with business meaning rather than stacking buzzwords.
- Keep this file generic. Preset-specific quirks such as banned sentence formulas or punctuation habits belong in the selected preset, not here.

## Structural logic

- Lead with the conclusion before expanding the argument.
- Organize sections as problem, mechanism, evidence, implication, and action when the topic allows it.
- Use comparison structures, frames, or tables when they improve scan speed.
- Make assumptions, boundary conditions, and unresolved questions explicit.
- End important sections with the decision implication or next move.

## Don'ts

- Do not slide into marketing slogans or inflated certainty.
- Do not pile up concepts without defining them and tying them to a business consequence.
- Do not fabricate precision when the evidence is weak.
- Do not use dense jargon chains that assume insider context unless the user clearly wants that voice.

## Acceptance checklist

- Does each section state the conclusion early?
- Are the main claims backed by named sources, or clearly marked as assumptions?
- Is the business implication explicit rather than implied?
- Does the report include at least one high-value comparison or structural frame where the topic benefits from it?
- Does the draft end with a next step, resource implication, or decision consequence instead of stopping at description?
